# 图片上传到阿里云服务器方案

## 文件结构

```
website-capital-final/
├── admin/
│   └── api/
│       └── upload-image.php    # 图片上传API
├── case-edit.html              # 修改后的案例编辑页面
└── setup-upload-dir.sh         # 服务器目录权限设置脚本
```

## 实现内容

### 1. PHP上传API (admin/api/upload-image.php)
- 接收图片文件上传
- 保存到服务器目录: `/www/wwwroot/47.95.236.85/uploads/`
- 返回图片URL: `http://47.95.236.85/uploads/文件名`
- 支持格式: jpg, png, gif, webp
- 最大文件大小: 5MB
- 检查目录写入权限

### 2. 修改后的case-edit.html
- **图片上传流程**: 选择图片 → 压缩 → 上传到服务器 → 存储URL
- **存储格式**: 只存储图片URL，不再存储Base64
- **前端压缩**: 压缩到最大1200x1200像素，质量80%
- **显示**: 直接使用服务器URL显示图片

### 3. 数据格式变化

**旧格式 (Base64)**:
```json
{
  "images": ["data:image/jpeg;base64,/9j/4AAQ..."]
}
```

**新格式 (URL)**:
```json
{
  "images": [{
    "id": "img_xxx",
    "name": "photo.jpg",
    "url": "http://47.95.236.85/uploads/20250427_xxx.jpg",
    "filename": "20250427_xxx.jpg",
    "size": 12345,
    "timestamp": 1745689200000
  }]
}
```

## 服务器部署步骤

### 1. 上传PHP文件
将 `admin/api/upload-image.php` 上传到服务器的 `/www/wwwroot/47.95.236.85/admin/api/` 目录

### 2. 创建上传目录并设置权限

```bash
# 在服务器上执行
mkdir -p /www/wwwroot/47.95.236.85/uploads
chown -R www:www /www/wwwroot/47.95.236.85/uploads
chmod -R 755 /www/wwwroot/47.95.236.85/uploads
```

或者使用提供的脚本:
```bash
bash setup-upload-dir.sh
```

### 3. 检查PHP配置
确保PHP允许上传足够大的文件:

```ini
upload_max_filesize = 5M
post_max_size = 8M
```

### 4. 上传修改后的case-edit.html
将修改后的 `case-edit.html` 上传到服务器

## 注意事项

1. **跨域问题**: API已设置CORS头，允许跨域访问
2. **旧数据兼容**: 页面仍兼容旧格式的Base64图片
3. **图片加载失败**: 添加了onerror处理，显示占位图
4. **存储空间**: 本地localStorage只存储URL，大大节省空间

## 优势

- ✅ 解决localStorage 5MB限制
- ✅ 图片永久存储在服务器
- ✅ 多设备可访问同一图片
- ✅ 页面加载更快（URL比Base64小）
- ✅ 支持更大的图片文件
