# 修改说明-修复页面删除功能

## 问题现象
在CMS后台页面管理中，所有页面的删除按钮点击后无任何反应。

## 根本原因

### 1. 系统页面按钮被 disabled 属性禁用
- `index.html` 中，所有 `defaultPages`（系统页面）的删除按钮都带 `disabled` 属性
- 带有 `disabled` 属性的 `<button>` 元素**不会触发 onclick 事件**
- 用户点击时没有任何反馈（没有提示、没有弹窗、没有任何视觉响应）

### 2. deletePage 函数只处理自定义页面
- 即使移除了 disabled 属性，`deletePage()` 也只从 `customPages` 数组中查找页面
- 系统页面的 `page` 为 `undefined`，会显示"系统页面不能删除"的提示并直接返回
- 也就是说系统页面完全无法被删除

### 3. dashboard.html 根本没有删除按钮
- 仪表盘页面列表只提供了编辑和查看按钮
- 没有渲染删除按钮，也没有对应的删除函数

## 修复方案

### 修改1：index.html - 删除按钮
**位置：** `renderPageList()` 函数中的按钮生成部分
**更改：**
- 移除 `disabled` 属性，让所有删除按钮都可点击
- 改为使用 `title` 属性提供提示信息
- 系统页面使用提示"系统页面，删除需谨慎"，自定义页面使用"删除此页面"

### 修改2：index.html - deletePage() 函数
**位置：** `deletePage()` 函数
**更改：**
- 查找范围从仅 `customPages` 扩展为 `[...defaultPages, ...customPages]` 所有页面
- 删除逻辑改为先尝试调用后端API，API成功后才执行本地删除
- API返回错误时，会向用户展示具体错误信息（如"系统页面受保护"）
- API请求失败时阻止本地删除，避免前后端数据不一致
- 修复了原来 API 失败仍继续本地删除的问题

### 修改3：dashboard.html - 添加删除功能
**位置：** `renderPageList()` + 新增函数
**更改：**
- 在列表模板中添加了删除按钮 `cms-page-btn-delete`
- 新增 `dashboardDeletePage()` 函数处理删除逻辑
- 添加了 `.cms-page-btn-delete` CSS 样式
- 新增 `showToast()` 函数用于显示操作反馈
